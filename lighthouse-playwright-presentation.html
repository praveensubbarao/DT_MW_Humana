The Problem

Without integration
  • Lighthouse runs manually or as a separate CI job
  • Performance regressions discovered late
  • Scores live in a separate dashboard — not next to test results
  • No easy way to assert on scores per build
With integration
  • Lighthouse runs as part of every yarn test
  • Full HTML report attached to the test result
  • Scores annotated per test — visible in Playwright report
  • Soft assertions → all categories checked, CI flags regressions

Key insight: Lighthouse launches its own headless Chrome via chrome-launcher on a separate port — no DevTools Protocol conflict with Playwright's browser.
Lazy by design: Lighthouse only runs in tests that call lighthouse.audit() or lighthouse.auditMobile(). Zero overhead for all other tests.

Step 1 — Core Runner NEW FILE

src/utils/lighthouse/runLighthouse.ts

export async function runLighthouse(
  url: string,
  formFactor: 'desktop' | 'mobile' = 'desktop',
): Promise<LighthouseResult> {
  const chrome = await launch({
    chromeFlags: ['--headless','--no-sandbox','--disable-gpu'],
  });
  const emulation = formFactor === 'mobile'
    ? { formFactor: 'mobile' as const,
        screenEmulation: { mobile: true, width: 412, height: 915,
                           deviceScaleFactor: 2.625, disabled: false } }
    : { formFactor: 'desktop' as const,
        screenEmulation: { mobile: false, width: 1350, height: 940,
                           deviceScaleFactor: 1, disabled: false } };
  try {
    const result = await lighthouse(url, {
      port: chrome.port, output: ['html','json'], logLevel: 'error',
      onlyCategories: ['performance','accessibility','best-practices','seo'],
      ...emulation,
    });
    // returns { scores: { performance, accessibility, bestPractices, seo }, htmlReport }
  } finally {
    await chrome.kill();   // always clean up — even on failure
  }
}

// Descriptive failure message — no guessing what threshold was missed
export function thresholdMsg(label, formFactor, score, threshold): string {
  const status = score >= threshold ? 'meets' : 'BELOW';
  return `${label} (${formFactor}): ${score} — ${status} industry threshold of ${threshold}`;
}

Step 3 — Playwright Fixture NEW FILE

src/utils/fixtures/lighthouseFixture.ts

export const lighthouseTest = base.extend<{ lighthouse: LighthouseActions }>({
  lighthouse: async ({}, use, testInfo) => {
    await use({
      audit: async (url) => {
        const result = await runLighthouse(url, 'desktop');

        // ① Attach full HTML report — clickable in Playwright dashboard
        await testInfo.attach('Lighthouse Report (desktop)', {
          body: result.htmlReport, contentType: 'text/html',
        });

        // ② Annotate scores with pass/fail status vs threshold
        testInfo.annotations.push(
          { type: 'lighthouse:desktop:performance',
            description: scoreStatus(scores.performance, DESKTOP_THRESHOLDS.performance) },
          // ... accessibility, best-practices, seo
        );
        return result.scores;
      },

      auditMobile: async (url) => {
        // Same — Lighthouse handles Pixel 5 emulation internally
        const result = await runLighthouse(url, 'mobile');
        // attach + annotate with MOBILE_THRESHOLDS
      },
    });
  },
});

Step 4 — The Test (Soft Assertions) NEW FILE

src/tests/performance/HomepageLighthouse.spec.ts

test('homepage desktop scores meet thresholds', async ({
  providerActions, lighthouse,
}) => {
  await providerActions.openProviderPortal();
  const scores = await lighthouse.audit(providerActions.getBaseURL() + '/');

  // expect.soft → ALL 4 categories checked — no early exit on first miss
  expect.soft(scores.performance,
    thresholdMsg('Performance','desktop',scores.performance,DESKTOP_THRESHOLDS.performance))
    .toBeGreaterThanOrEqual(DESKTOP_THRESHOLDS.performance);

  expect.soft(scores.accessibility, /* ... */).toBeGreaterThanOrEqual(90);
  expect.soft(scores.bestPractices, /* ... */).toBeGreaterThanOrEqual(90);
  expect.soft(scores.seo,           /* ... */).toBeGreaterThanOrEqual(90);
});

// Mobile counterpart uses lighthouse.auditMobile(url) + MOBILE_THRESHOLDS

import { mergeTests }              from '@playwright/test';

Key Takeaways

🔀 No CDP Conflict

Lighthouse uses its own Chrome via chrome-launcher. Playwright's browser is completely unaffected.

📎 Report in the Dashboard

testInfo.attach() embeds the full HTML report as a clickable artifact in the Playwright HTML dashboard.

📱 Desktop + Mobile

Pass formFactor: 'mobile' to Lighthouse for Pixel 5 emulation — independent of any Playwright device project.

🛡️ Soft Assertions

expect.soft() reports all four categories every run. Descriptive messages show exactly which threshold was missed and by how much.

⚡ Lazy by Design

Lighthouse only runs in tests that call audit(). Zero overhead for the rest of the suite.

📐 Industry Thresholds

Thresholds based on Google's 0–49 / 50–89 / 90–100 scoring bands. Shared constants between fixture annotations and test assertions.
